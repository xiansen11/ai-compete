# Ragent 模块详细关系图

本文档详细描述 Ragent AI 项目中各个模块之间的关系、依赖调用链路和数据流向。

---

**依赖方向说明:**
- ► `bootstrap` 依赖 `framework` + `infra-ai` (编译期 + 运行期)
- ► `infra-ai` 依赖 `framework` (编译期 + 运行期)
- ► `mcp-server` 独立部署，通过 HTTP 与 `bootstrap` 通信 (运行期)

      │

## 七、模块边界与通信协议总结

| 模块对 | 通信方式 | 协议/技术 | 调用场景 |
|--------|---------|----------|---------|
| **bootstrap → framework** | 方法调用 | Java (同一 JVM) | 所有业务逻辑 |
| **bootstrap → infra-ai** | 方法调用 | Java (同一 JVM) | AI 模型调用 |
| **bootstrap → mcp-server** | HTTP | JSON-RPC 2.0 | MCP 工具调用 |
| **bootstrap → Milvus** | gRPC/HTTP | Milvus SDK | 向量检索 |
| **bootstrap → MySQL** | JDBC | MyBatis Plus | 数据持久化 |
| **bootstrap → Redis** | TCP | Redisson | 缓存/限流 |
| **bootstrap → RocketMQ** | TCP | RocketMQ SDK | 异步消息 |
| **infra-ai → 外部 AI** | HTTP | OpenAI API 风格 | 模型调用 |

## 八、核心设计理念

1. **分层清晰**：framework（技术）/ infra-ai（AI）/ bootstrap（业务）职责分离
2. **依赖倒置**：上层依赖下层接口，便于测试和替换
3. **松耦合**：mcp-server 独立部署，通过标准协议通信
4. **高可用**：infra-ai 层提供熔断降级，单点故障不影响全局
5. **可扩展**：新增模型/工具只需扩展对应模块，不影响现有代码
